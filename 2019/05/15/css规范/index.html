记录一些css习惯

一些约定语法

  • 使用两个空格代表Tab --保证所有环境下一致的展现
  • 选择器分组，将单独的选择器放在一行。
  • 每个声明块的左花括号前面添加一个空格，声明块的右花括号单独成行 --代码易读性。
  • 每个属性名与之后的:之间不允许有空格。
  • 每个声明的语句 ：后面应该插入一个空格。
  • 每一个声明单独占一行，-- 获得准确的错误报告。
  • 每一行不得超过120个字符。-- 除非单行不可分割，例如URL超长。
  • 声明语句都以分号结尾，最后的一个分号是可选的，但是省略容易出错。
  • 以逗号分隔的属性值，没个逗号后面应该插入空格。
  • 不要在rgb(),rgba(),hsla()或者rect()值的内部的逗号后面插入空格，利于从多个属性值（既加逗号也加空格）中区分多个颜色值。
  • 属性值或颜色参数，省略小于1的小数前面的0（例如： .5px代替 0.5px）。
  • 十六进制值应该全部小写，例如：#fff。扫描文档的时候小写字符更容易分辨，并且尽量用简写的形式，例如：#fff代替#ffffff。
  • 选择器中的属性添加双引号，例如 input[type="text"],代码的一致性。
    -设置长度单位时 0值不设置单位。

字体

  • font-family属性中的字体族名称应使用字体的英文Family Name，其中有空格，需放置在引号中。
  • font-family按[英文字体在前，中文字体在后]，[效果佳 (质量高/更能满足需求)的字体在前、效果一般的字体在后]顺序编写，最后必须指定一个通用字体族( serif / sans-serif )。
  • font-family不区分大小写，但是在同一个项目中，大小写应该一致.
  • 需要在Windws平台显示的中文内容，字号应该不小于12px。
  • 需要Windows平台显示中文内容，不要使用除normal意外的font-style,其他平台也是谨慎使用。
  • font-weight属性使用数值方式描述。

  • line-height定义文本段落时，使用数值。
    将line-height，设置为数值，浏览器会基于当前元素设置的 font-size进行再次计算。在不同字号的文本段落组合中，能达到较为舒适的行间间隔效果，避免在每个设置了 font-size 都需要设置 line-height。

    当 line-height 用于控制垂直居中时，还是应该设置成与容器高度一致

声明顺序

属性的声明顺序如下：

  1. Postioning (布局方式，位置)
  2. Box model (尺寸)
  3. Typographic (文本相关)
  4. Visual (视觉效果)

第一个Postioning可以从正常文档流中移除元素，而且能覆盖盒模型的一些样式，第二个box model 盒模型决定了租家的尺寸和位置。

而之后的字体设置和视觉相关属性的是影响内部的所以排在后面。

  • Pormatting 相关属性：postion/top/right/bottom/left/float/diaplay/overflow等。
  • Box Model 相关属性：border/margin/padding/width/height等。
  • Typographic 相关属性：font/line-height/text-align/word-wrap等。
  • Visual相关属性： background/color/transtion/list-style等。

    清除浮动

    通过伪类设置clear或者出发BFC的方式进行clearfix,避免增加空标签的方式。
    触发BFC方式：
  • float非none
  • position 非 static
  • overflow 非 visible

    z-index

    将z-index进行分层，对文档流外绝对定位元素的视觉层级关系进行管理。
