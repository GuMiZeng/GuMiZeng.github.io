区分sessionStirage、localStorage、Cookie、Session

本地存储

sessionStirage 和 localStorage

方法：
- .key = value        赋值

- .setItem(key,value) 赋值

- .getItem(key)       取值

- .removeItem(key)    移除值

- .clear() 删除所有的sessionStorage

区别： 

- sessionStorage是关闭窗口数据消失，每个窗口的数据不能共享，但是前进后退，数据不会丢失
- localSorage不主动删除就持久的保存数据，在某一个域下，localStorage是多个窗口共享

cookie:

1. Cookie通过在客户端记录信息确定用户身份
2. 会话Cookie和持久Cookie
    - 若不设置过期时间，则表示这个cookie的生命期为浏览器会话期间，关闭浏览器窗口，cookie就消失。这种生命期为浏览器会话期的cookie被称为会话cookie。会话cookie一般不存储在硬盘上而是保存在内存里，当然这种行为并不是规范规定的。

    - 设置了过期时间，浏览器就会把cookie保存到硬盘上，关闭后再次打开浏览器，这些cookie仍然有效直到超过设定的过期时间。存储在硬盘上cookie可以在浏览器的不同进程间共享。这种称为持久Cookie。 

3. Cookie具有不可跨域名性

就是说，浏览器访问百度不会带上谷歌的cookie.

session:

1. Session通过在服务器端记录信息确定用户身份.
2. 判断状态：session ID，由cookie传递到服务器
    - 不使用cookie，使用url重写——sessionID放在url路径后
    - 使用表单隐藏字段
3. Session 共享

在各自的cookie中，因此服务器会认为对于两个子站的访问,是来自不同的会话。
解决的方法是通过修改cookies的域名为父域名达到cookie共享的目的,从而实现SessionId的共享。带来的弊端就是，子站间的cookie信息也同时被共享了。
